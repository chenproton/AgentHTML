# BrochureCraft 模板制作规范

以下是制作 **BrochureCraft 可导入模板** 的完整规范，保存这份文档，做模板时逐条对照即可。

---

## 一、文件格式

| 格式 | 支持 | 说明 |
|------|------|------|
| `.html` | ✅ | 单文件 HTML，可直接导入 |
| `.zip` | ✅ | 组合导入：同时上传封面 + 内页两个 HTML |

### 1.1 单 HTML 导入
- 上传一个 `.html` 文件时，系统按内容自动识别：
  - 含 `.a4-page` → 作为**封面模板**
  - 含 `.page` 且无 `.a4-page` → 作为**内页模板**

### 1.2 ZIP 组合导入（推荐）
- ZIP 内放两个 HTML 文件：
  - `cover.html`：封面模板（必须含 `.a4-page`）
  - `inner.html`：内页模板（必须含 `.page`）
- 命名符合上述规则时系统优先识别；不符合时也会自动按内容识别。

---

## 二、两种模板类型

BrochureCraft 同时支持**封面模板**和**内页模板**。两者结构不同，请勿混用。

| 类型 | 根容器 | 尺寸 | 用途 |
|------|--------|------|------|
| 封面 | `.a4-page` | 210mm × 297mm（A4） | 企业封面、章节页、尾页等 |
| 内页 | `.page` | 默认 9920px × 6732px | 岗位 / 单元 / 知识点环形布局 |

---

## 三、封面模板结构

### 3.1 页面容器标记（必填）

每个 A4 页面必须用这个结构包裹：

```html
<div class="a4-page" data-page-id="cover" data-page-name="封面">
  <!-- 页面内容 -->
</div>
```

| 属性 | 必填 | 说明 |
|------|------|------|
| `class="a4-page"` | ✅ | 必须，系统靠这个识别页面边界 |
| `data-page-id` | ✅ | 唯一标识，英文，如 `cover`、`p1`、`p2` |
| `data-page-name` | ✅ | 显示在左侧边栏的中文名称 |

**注意**：不要嵌套 `.a4-page`，每个页面独立一个容器。

### 3.2 A4 尺寸 CSS 规范（必须包含）

模板 `<head>` 中必须包含以下基准样式，确保 A4 尺寸正确：

```html
<style>
  @page { size: A4; margin: 0; }

  * { margin: 0; padding: 0; box-sizing: border-box; }

  body {
    font-family: "PingFang SC", "Microsoft YaHei", sans-serif;
    background: #f0f0f0;
  }

  .a4-page {
    width: 210mm;
    height: 297mm;
    padding: 20mm;
    box-sizing: border-box;
    page-break-after: always;
    background: white;
    position: relative;
    overflow: hidden;
  }

  .a4-page:last-child {
    page-break-after: auto;
  }
</style>
```

**注意**：
- 系统导入时会强制注入 `!important` 覆盖，但模板本身也应写正确
- `padding` 可以根据设计需求调整（如 `0` 做全出血设计，或 `30mm` 做宽边距）
- 不要给 `.a4-page` 写 `display: flex` 等可能干扰分页的属性（可以写在内部子容器上）

## 四、内页模板结构

### 4.1 页面容器标记（必填）

内页模板只有一个根容器：

```html
<div class="page" style="width: 9920px; height: 6732px;">
  <!-- 岗位 / 单元 / 知识点内容 -->
</div>
```

| 属性 | 必填 | 说明 |
|------|------|------|
| `class="page"` | ✅ | 必须，系统靠这个识别为内页模板 |
| `style="width: ...; height: ..."` | ✅ | 内页画布尺寸，建议 `9920px × 6732px` |

### 4.2 内页布局结构（必须遵循）

内页采用「岗位卡片 + 时间轴单元 + 环形知识点」布局，系统通过固定 class 识别结构并支持增删改。请严格使用以下类名：

```html
<div class="page" style="width: 9920px; height: 6732px;">
  <div class="cards-panel">
    <div class="cards-row">
      <!-- 岗位卡片 1 -->
      <div class="job-card">
        <div class="job-card__title" data-field="job_0" data-type="text">岗位名称</div>
        <div class="job-card__desc" data-field="job_0_desc" data-type="text">岗位描述</div>
        <ul class="tools-list">
          <li class="tools-list__item" data-field="job_0_tool_0" data-type="text">工具一</li>
        </ul>
        <div class="job-card__timeline">
          <svg class="timeline__connections">...</svg>
          <div class="timeline-unit">
            <div class="timeline-unit__node">
              <span data-field="job_0_unit_0" data-type="text">单元名称</span>
            </div>
            <div class="timeline-unit__capsule" data-field="job_0_unit_0_capsule" data-type="text">100课时</div>
            <div class="timeline-unit__point timeline-unit__point--tl" data-field="job_0_unit_0_point_0" data-type="text">知识点</div>
          </div>
        </div>
      </div>
      <!-- 更多岗位卡片 ... -->
    </div>
  </div>
</div>
```

**关键 class 说明**：

| class | 作用 | 约束 |
|-------|------|------|
| `.cards-panel` | 岗位卡片总容器 | 一个内页必须有且仅有一个 |
| `.cards-row` | 前两个岗位卡片容器 | 系统会自动管理 |
| `.cards-grid` | 第三个及以上岗位卡片容器 | 系统会自动创建，模板里可不写 |
| `.job-card` | 单个岗位卡片 | 最多支持 8 个 |
| `.job-card__title` | 岗位名称 | 必填，用于结构面板识别 |
| `.job-card__desc` | 岗位描述 | 建议保留 |
| `.tools-list` / `.tools-list__item` | 训练工具列表 | 建议保留 |
| `.job-card__timeline` | 单元时间轴容器 | 必填 |
| `.timeline-unit` | 单个单元 | 每岗位最多 7 个 |
| `.timeline-unit__node` / `span` | 单元名称 | 必填 |
| `.timeline-unit__capsule` | 单元课时/标签 | 建议保留 |
| `.timeline-unit__point` | 知识点标签 | 每单元最多 4 个，位置由 `--tl/--tr/--bl/--br` 控制 |

**注意**：
- 知识点位置类名必须是 `timeline-unit__point--tl`（左上）、`--tr`（右上）、`--bl`（左下）、`--br`（右下）四选一
- 系统会自动对 `data-field` 进行重编号（`job_0`、`job_0_unit_0`、`job_0_unit_0_point_0` 等），模板中可先用任意唯一名称
- 新增/删除岗位、单元、知识点后，系统会自动重新布局时间轴和连接线

### 4.3 内页 CSS 规范

内页尺寸较大，建议使用固定像素单位，并确保：

```html
<style>
  * { margin: 0; padding: 0; box-sizing: border-box; }

  .page {
    width: 9920px;
    height: 6732px;
    position: relative;
    overflow: hidden;
    background: white;
  }

  /* 岗位卡片示例样式 */
  .job-card {
    width: 1200px;
    position: relative;
  }

  .timeline-unit {
    position: absolute;
    width: 480px;
    height: 480px;
  }

  .timeline-unit__point {
    position: absolute;
  }

  .timeline-unit__point--tl { top: 0; left: 0; }
  .timeline-unit__point--tr { top: 0; right: 0; }
  .timeline-unit__point--bl { bottom: 0; left: 0; }
  .timeline-unit__point--br { bottom: 0; right: 0; }
</style>
```

## 五、可替换字段标记（必填）

在页面容器内，对需要替换的元素添加 `data-field` 和 `data-type`。

系统仅支持两种字段类型：**纯文本** 和 **图片**。

### 5.1 纯文本

```html
<h1 data-field="main_title" data-type="text">默认标题</h1>
<p data-field="subtitle" data-type="text">默认副标题</p>
<span data-field="page_number" data-type="text">01</span>
```

- 替换时只替换纯文本内容（`textContent`）
- HTML 结构和样式（字体、颜色、大小等）完全保留

### 5.2 图片

```html
<img data-field="logo_img" data-type="image" src="data:image/png;base64,..." alt="Logo">

<!-- 也支持 div 背景图风格（Pencil 设计工具导出格式） -->
<div data-field="banner_bg" data-type="image" style="background-image: url(...)"></div>
```

- 替换时同时修改 `src` 属性和 `style.backgroundImage`
- 支持 `http://` URL、`data:image` base64、或上传本地文件

### 字段类型速查

| `data-type` | 替换方式 | 适用场景 |
|-------------|----------|----------|
| `text` | `textContent` | 所有文字内容（标题、段落、数字、单元名、知识点等） |
| `image` | `src` + `backgroundImage` | Logo、产品图、背景图 |

### 5.3 多个不同样式的字段要拆开标记

如果一个字段区域内包含**样式不同的多个子元素**（如数字编号 + 文字说明），必须为每个子元素单独标记，不要用一个 `data-field` 包住全部。

```html
<!-- ❌ 错误：一个 field 包住两个不同样式的 span，编辑后会丢失样式 -->
<div class="biz-item" data-field="biz_01" data-type="text">
  <span class="biz-num">01</span>
  <span class="biz-text">智慧物流</span>
</div>

<!-- ✅ 正确：每个样式独立的子元素各自标记 -->
<div class="biz-item">
  <span class="biz-num" data-field="biz_01_num" data-type="text">01</span>
  <span class="biz-text" data-field="biz_01_text" data-type="text">智慧物流</span>
</div>
```

**原则**：`data-field` 应标记在**最细粒度的样式叶子节点**上，确保每个字段只包含纯文本（无嵌套 HTML），这样编辑后样式不会丢失。

## 六、资源处理规范

### 6.1 图片（推荐 base64 内联）

```html
<img src="data:image/png;base64,iVBORw0KGgoAAAANSUhEUgAA...">
```

**不推荐**外部 URL，因为：
- 预览时可能跨域失败
- 导出 PDF 时可能加载不到
- 离线环境无法使用

### 6.2 字体（推荐网络字体 CDN）

```html
<link href="https://fonts.googleapis.com/css2?family=Noto+Sans+SC:wght@400;700&display=swap" rel="stylesheet">
```

或直接用系统安全字体：
```css
font-family: "PingFang SC", "Microsoft YaHei", "Helvetica Neue", Arial, sans-serif;
```

### 6.3 CSS（必须内联）

```html
<style>
  /* 所有样式写在这里 */
</style>
```

**不支持**外部 CSS 文件：
```html
<!-- ❌ 不要这样写，导入后样式会丢失 -->
<link rel="stylesheet" href="styles.css">
```

### 6.4 JavaScript

模板中**不要写 JavaScript**。系统导入时会忽略 `<script>` 标签，且 JS 在 PDF 导出中不生效。

## 七、AI 编辑提示规范（供参考）

当用户通过右侧 AI 助手修改模板时，AI 会遵循以下规则：

- 文案修改 → 通过 `replacements` 数组替换字段值，不破坏 HTML 结构
- 样式修改 → 优先返回 CSS，尽量少返回 HTML
- 修改页面背景色时：
  - 封面：targeting `.a4-page` 或其功能类（`.cover` / `.content`）
  - 内页：targeting `.page`
  - 使用 `background` 简写并加 `!important`，例如：
    ```css
    .a4-page.cover { background: #E6F3FF !important; }
    ```
  - 禁止只改 `background-color`，禁止改嵌套文字块/图片容器的背景

## 十、注意事项清单

| ✅ 应该做 | ❌ 不要做 |
|----------|----------|
| 封面用 `.a4-page`，内页用 `.page` | 把 `.a4-page` 和 `.page` 混在同一个模板里 |
| CSS 全部写在 `<style>` 标签内 | 引用外部 `.css` 文件 |
| 图片用 base64 或 SVG 内联 | 引用外部图片 URL（可能跨域/加载失败）|
| 字体用系统字体或 CDN 网络字体 | 依赖本地安装的字体 |
| 每个封面页面独立一个 `.a4-page` 容器 | 嵌套 `.a4-page` 或一个容器放多页 |
| 内页使用规定的 class 结构（`.job-card`、`.timeline-unit` 等） | 随意改内页 class 名，否则增删功能失效 |
| 保留 `data-field` 和 `data-type` 属性 | 删除或改名这些属性 |
| 保持 `data-page-id` 唯一 | 多个页面用相同的 `data-page-id` |
| 用安全字体栈确保兼容性 | 只用某一种特殊字体 |
| 测试时直接用浏览器打开 HTML 看效果 | 不做测试直接导入 |
| 不同样式的子元素各自单独标记 `data-field` | 用一个大 `data-field` 包住多种不同样式的元素 |

## 十一、快速自检

模板做好后，用浏览器直接打开 HTML 文件，检查：

1. [ ] 封面每个页面都是 **210mm × 297mm** 的竖向长方形
2. [ ] 内页 `.page` 尺寸正确（默认 9920px × 6732px）
3. [ ] 页面之间有明显的分页间隔（封面）
4. [ ] 所有 `data-field` 元素都能在页面上找到
5. [ ] 没有引用任何外部文件（除了可选的字体 CDN）
6. [ ] 图片都能正常显示（不是裂图）
7. [ ] ZIP 组合导入时文件名符合 `cover.html` + `inner.html`

全部通过后再导入 BrochureCraft 使用。
